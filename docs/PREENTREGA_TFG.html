Descripción del proyecto

Sabor de Casa es una aplicación multiplataforma Android Web para un local de comida preparada para llevar con servicio de catering para eventos. Desarrollada como Proyecto Final del Grado Superior DAM.

El negocio gestiona cuatro tipos de pedido: venta directa en mostrador, recogida en local, pedidos a domicilio y encargos para fecha futura. Además ofrece un servicio de catering para eventos privados (bodas, comuniones, empresas, etc.).

Funcionalidades implementadas

  • Autenticación: registro/login con Supabase Auth, roles client, employee y admin
  • Menú y carrito: catálogo con filtros, carrito persistente, proceso de checkout
  • Pedidos: historial de pedidos y seguimiento de estado en tiempo real (Realtime)
  • Encargos y catering: formulario de solicitud con selección de fecha, hora y menú personalizado
  • Panel de cocina: vista de empleado con cola de pedidos activos y cambio de estado
  • Reparto: pantalla de repartidor con listado de pedidos a domicilio
  • TPV mostrador: terminal punto de venta para venta directa sin cuenta
  • Panel de administración: dashboard con métricas, gestión de usuarios, productos, pedidos y configuración
  • Chat IA: asistente integrado para consultas sobre el menú y el local
  • Notificaciones por email: confirmación de registro y seguimiento de pedidos (Brevo)
  • Notificaciones push: alertas de cambio de estado del pedido vía FCM
  • Pagos: integración con Stripe para pago con tarjeta en móvil y web

Arquitectura

La aplicación sigue una arquitectura en capas (UI → Logic → Data) con separación por features: 

lib/
├── core/           # Tema, router, widgets compartidos, utilidades
├── features/       # Módulos independientes (auth, menu, cart, orders, admin…)
└── services/       # Capa de acceso a Supabase

El backend se apoya en Supabase: base de datos PostgreSQL con Row Level Security, funciones PL/pgSQL para lógica de negocio, y Edge Functions en Deno para integraciones externas (Stripe, Brevo, FCM).
