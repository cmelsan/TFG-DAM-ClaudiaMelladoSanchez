1. Requisitos previos

1.1 Hardware mínimo recomendado para desarrollo

  • CPU: x64 de 4 núcleos (Intel i5 / Ryzen 5 o superior).
  • RAM: 16 GB (mínimo 8 GB, con limitaciones evidentes).
  • Disco: 30 GB libres en SSD.
  • GPU: cualquier integrada moderna; obligatoria para emuladores Android.

2. Instalación del entorno

2.1 Instalar Flutter

Seguir la guía oficial en https://docs.flutter.dev/get-started/install/windows. Verificar:

flutter --version
flutter doctor -v

Deben aparecer todos los apartados con tick verde (Android toolchain, Chrome, VS Code).

2.2 Clonar el repositorio

git clone https://github.com/<tu-usuario>/sabor-de-casa.git
cd sabor-de-casa
flutter pub get

2.3 Generar código (Riverpod, Freezed, JsonSerializable)

dart run build_runner clean
dart run build_runner build --delete-conflicting-outputs

2.4 Variables de entorno

Crear el archivo .env.development en la raíz con la siguiente plantilla:

SUPABASE_URL=https://vrxliepwzvdrcxpdgpnd.supabase.co
SUPABASE_ANON_KEY=<anon-key>
STRIPE_PUBLISHABLE_KEY=pk_test_xxx
STRIPE_MERCHANT_COUNTRY=ES

Importante: no commitear los archivos .env.*. Están listados en .gitignore. La service_role_key NUNCA debe llegar al cliente, solo se usa en Edge Functions.

3. Configuración de Supabase

3.1 Crear el proyecto

  1. Acceder a https://app.supabase.com y crear un nuevo proyecto.
  2. Anotar la URL del proyecto y la anon key.
  3. Crear los buckets de Storage necesarios (menu-images, profile-avatars) con políticas públicas de lectura.

3.2 Aplicar migraciones

Las migraciones SQL viven en supabase/migrations/ (28 archivos numerados 00001 - 00028). Por razones de desfase de historial documentadas, se aplican manualmente:

  1. Abrir el SQL Editor en Supabase Studio.
  2. Ejecutar en orden cada archivo SQL.
  3. Verificar tras cada bloque que no haya errores (tablas, índices y políticas creadas).

3.3 Configurar secretos de Edge Functions

supabase secrets set STRIPE_SECRET_KEY=sk_test_xxx
supabase secrets set STRIPE_WEBHOOK_SECRET=whsec_xxx
supabase secrets set BREVO_API_KEY=xkeysib-xxx
supabase secrets set BREVO_SENDER_EMAIL=no-reply@sabordecasa.local
supabase secrets set BREVO_SENDER_NAME="Sabor de Casa"
supabase secrets set FCM_SERVER_KEY=AAAA...

3.4 Desplegar las 8 Edge Functions

supabase functions deploy create-payment-intent
supabase functions deploy stripe-webhook --no-verify-jwt
supabase functions deploy send-order-notification
supabase functions deploy send-encargo-confirmation
supabase functions deploy send-encargo-reminders
supabase functions deploy send-catering-notification
supabase functions deploy send-newsletter
supabase functions deploy send-welcome-email
supabase functions deploy chat-bot

4. Configuración de servicios externos

4.1 Stripe

  • En el dashboard de Stripe, crear las claves test y live.
  • Configurar un endpoint webhook apuntando a https://<project>.functions.supabase.co/stripe-webhook con los eventos: payment_intent.succeeded, payment_intent.payment_failed.
  • Copiar el signing secret al secreto STRIPE_WEBHOOK_SECRET.

4.2 Brevo (emails)

  • Crear una API key con permisos Transactional Email.
  • Verificar el dominio remitente: añadir registros SPF, DKIM y, opcionalmente, DMARC.
  • En desarrollo se puede usar el remitente Brevo por defecto.

4.3 Firebase Cloud Messaging

  1. Crear un proyecto Firebase y registrar la app Android con el applicationId del proyecto (com.sabordecasa.app).
  2. Descargar google-services.json y colocarlo en android/app/.
  3. Activar Cloud Messaging y copiar el server key al secreto FCM_SERVER_KEY de Supabase.

  • r en la consola → hot reload (mantiene estado).
  • R en la consola → hot restart (reinicia el estado).

5.4 Análisis estático

dart analyze

Debe devolver 0 errors. Los warnings y los info son tolerables si no afectan a compilación.

6. Compilación para producción

6.1 Build Web

flutter build web --release \
  --dart-define-from-file=.env.production \
  --base-href "/"

Salida: build/web/. Subir todo el contenido al hosting estático.

6.2 Build Android (APK)

flutter build apk --release \
  --dart-define-from-file=.env.production

Salida: build/app/outputs/flutter-apk/app-release.apk.

6.3 Build Android (App Bundle para Play Store)

flutter build appbundle --release \
  --dart-define-from-file=.env.production

Salida: build/app/outputs/bundle/release/app-release.aab.

El bundle debe firmarse con un keystore propio configurado en android/key.properties antes del primer release. NO usar el debug.keystore en producción.

7. Despliegue

7.1 Web (Vercel)

  1. Crear un proyecto en Vercel apuntando a la carpeta build/web como output.
  2. Configurar como framework preset: Other.
  3. Build command: flutter build web --release --dart-define-from-file=.env.production.
  4. Output directory: build/web.
  5. Definir las variables de entorno necesarias en el panel de Vercel.

7.2 Android (Play Store)

  1. Subir el app-release.aab a la consola de Play Store.
  2. Completar fichas: descripción, capturas, política de privacidad.
  3. Configurar canal interno → cerrado → producción.

8. Mantenimiento

8.1 Tareas periódicas recomendadas

  • Semanal: revisar errores de Sentry / logs de Supabase y atender pedidos atascados en estados intermedios.
  • Mensual: actualizar dependencias con flutter pub upgrade --major-versions y volver a ejecutar build_runner.
  • Trimestral: rotar claves de Stripe / Brevo / FCM y refrescar SPF/DKIM si cambia el remitente.
  • Backups: Supabase realiza snapshots automáticos diarios. Exportar manualmente con pg_dump antes de migraciones críticas.

8.2 Regenerar codegen tras cambios en modelos

dart run build_runner clean
dart run build_runner build --delete-conflicting-outputs

8.3 Aplicar una nueva migración

  1. Crear el archivo en supabase/migrations/000XX_nombre.sql.
  2. Probarla localmente o en un entorno de staging.
  3. Aplicarla en Supabase Studio (SQL Editor) en producción.
  4. Documentarla en docs/SUPABASE.md.
