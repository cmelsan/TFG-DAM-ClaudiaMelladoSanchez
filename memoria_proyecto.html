Resumen ejecutivo

Sabor de Casa es una aplicación multiplataforma (Android nativa + Web responsive) desarrollada con Flutter 3.38 y respaldada por un backend serverless construido sobre Supabase. Soluciona la necesidad de un restaurante familiar de digitalizar todo su flujo de negocio: catálogo, pedidos, pagos online, catering, comunicación con clientes y gestión interna.

El proyecto entrega un producto funcional, seguro y desplegable: 41 pantallas en producción, 28 migraciones de base de datos, 8 funciones serverless y 3 roles diferenciados con permisos garantizados a nivel de base de datos mediante Row Level Security.

1. Introducción y motivación

El proyecto nace de una necesidad real observada en un restaurante familiar de carácter tradicional: la gestión de pedidos para llevar y de encargos para eventos se realizaba telefónicamente, lo que generaba errores, pérdida de pedidos y una pésima trazabilidad. La presencia digital se reducía a una página de Facebook desactualizada.

Como alumna del CFGS DAM, identifiqué este contexto como una oportunidad perfecta para aplicar de forma integrada todos los conocimientos adquiridos durante el ciclo: desarrollo multiplataforma con Flutter, bases de datos relacionales, servicios web, autenticación, pagos en línea, notificaciones push, seguridad y despliegue.

La motivación principal ha sido entregar un producto real, no un prototipo académico: una aplicación capaz de funcionar en producción desde el primer día, con un nivel de calidad equiparable al de soluciones comerciales.

2. Objetivos del proyecto

2.1 Objetivos generales

  • Diseñar y desarrollar una aplicación multiplataforma (Android + Web) usable en producción.
  • Cubrir el ciclo completo de un pedido, desde la captación del cliente hasta la entrega.
  • Implementar la seguridad como pilar transversal, no como añadido.
  • Producir documentación profesional cumpliendo la guía oficial del proyecto final DAM.

2.2 Objetivos específicos

  • Carta digital gestionable por el administrador desde la propia app.
  • Carrito y checkout con pago en línea integrado.
  • Tres roles (cliente, empleado, administrador) con paneles dedicados.
  • Notificaciones push y emails transaccionales automáticos.
  • Sistema de catering, soporte interno, newsletter y testimonios.
  • Despliegue real y replicable mediante manuales técnicos.

3. Marco tecnológico

La elección tecnológica responde a tres criterios: productividad de la persona desarrolladora individual, cobertura multiplataforma sin duplicar código, y coste cero o casi cero en infraestructura.

3.1 Frontend

  • Flutter 3.38 / Dart 3.5 como framework multiplataforma.
  • Riverpod 2.6 con codegen para gestión reactiva del estado.
  • Freezed + json_serializable para modelos inmutables y serialización Supabase.
  • GoRouter para navegación declarativa.
  • Material 3 con tema personalizado de marca.

3.2 Backend (BaaS)

  • Supabase: PostgreSQL gestionado, Auth, Storage, Realtime y Edge Functions (Deno).
  • Row Level Security (RLS) activado en todas las tablas con datos sensibles.

3.3 Servicios externos

  • Stripe — pagos con tarjeta.
  • Brevo — email transaccional y newsletter.
  • Firebase Cloud Messaging — notificaciones push.

4

5

Seguimiento y evaluación

Tutorías semanales, registro de incidencias, medición de KPIs, recogida de feedback de usuarios reales y lecciones aprendidas.

6.3 Calidad y mantenibilidad

  • Código organizado por features con separación estricta de capas.
  • Modelos inmutables con codegen, evitando errores de mutación accidental.
  • Seguridad centralizada en la base de datos: el frontend no puede saltarse permisos.
  • Documentación exhaustiva que permite a una nueva persona desarrolladora ponerse al día en 1 día.

7. Conclusiones

El proyecto ha cumplido y superado todos los objetivos planteados. Sabor de Casa es hoy una aplicación lista para producción que aporta valor inmediato al restaurante real para el que fue concebida.

A nivel personal y formativo, el proyecto ha exigido integrar todos los módulos del ciclo formativo en una única solución coherente. La combinación de Flutter con Supabase ha demostrado ser una elección sólida para una persona desarrolladora individual con ambición de cobertura multiplataforma.

Los retos más relevantes han sido la curva de aprendizaje del codegen de Riverpod, la gestión de seguridad mediante RLS y la integración cross-platform de Stripe. Cada uno ha aportado conocimiento aplicable a proyectos futuros, dentro y fuera del entorno académico.

La principal conclusión es que el rol de la persona desarrolladora moderna no se limita a escribir código: planificar, documentar, comunicar avances y evaluar críticamente el propio trabajo son competencias igual de relevantes y han sido especialmente trabajadas en este proyecto.
